Why this exists

The Silent Failure Problem

Without This System

A developer spends 4–8 hours per audit cycle cross-referencing flagged violations against the 78-page WCAG specification. Tools like axe-core tell you that something is wrong — not how to fix it. A base LLM asked to audit a URL generates confident-looking violations it didn't actually detect from your HTML. Its fixes can produce aria-label="icon-button-14" — syntactically valid, semantically meaningless. The violation is invisible now, which is worse.

With This System

A dedicated input layer works on real code — fetching live HTML from URLs, reading raw HTML, or reconstructing it from screenshots. A RAG layer retrieves the actual W3C specification text before generating any analysis — the model reads it, not its memory of it. Every fix is annotated with the specific WCAG Success Criterion it satisfies (<!-- FIXED: 1.1.1 -->) and validated by re-audit. Target silent failure rate: < 5%.

What it does

Key Features

🌐

Live URL Crawling

Enter any public URL — the system fetches the real HTML and audits it directly. No copy-pasting required. This is the proposal's core differentiator: working on actual code, not a hallucinated version of it.

🔍

WCAG 2.2 Violation Detection

Identifies violations across all four WCAG principles — Perceivable, Operable, Understandable, Robust — with severity ratings and pinpoints the exact offending element. Grounded in the actual W3C specification via RAG.

✨

Automated Fix Generation

Every violation is auto-fixed with annotated HTML comments (<!-- FIXED: 1.1.1 -->) citing the specific WCAG Success Criterion. A side-by-side code diff shows exactly what changed and why.

📸

Multimodal Screenshot Input

Upload any webpage screenshot. LLaMA 4 Scout Vision (Groq) reconstructs the HTML structure from the image, then the full audit pipeline runs — no source code access needed.

📚

RAG — WCAG Knowledge Base

A ChromaDB vector store with 42 WCAG 2.2 criteria is queried semantically before each audit. Top-3 relevant criteria are injected into every prompt — the model can't invent a standard it's actively reading.

📊

Evaluation Metrics Panel

Fix Validity Rate (target ≥ 90%), WCAG Pass Rate (target ≥ 85%), and Silent Failure Target (< 5%) displayed after every audit — directly from the proposal's evaluation strategy.

3 Core AI Components Implemented

Component 1

Retrieval-Augmented Generation (RAG)

42 WCAG 2.2 criteria are chunked, embedded with ChromaDB's built-in sentence-transformers model, and stored in a local vector store. Each audit retrieves the top-3 semantically relevant criteria via cosine similarity and injects them into the prompt — grounding the model's analysis in the official W3C specification rather than training-data memory. No external embedding API required.

ChromaDB sentence-transformers Cosine Similarity Semantic Search
Component 2

Multi-Step Prompt Engineering

Four sequential prompts with explicit JSON output schemas, system role assignment, RAG context injection, and temperature control (0.0 for deterministic detection/fixing, 0.2 for explanations). The chain re-audits fixed HTML automatically to compute fix validity metrics that match the proposal's evaluation targets.

Groq API llama-3.3-70b-versatile JSON Schema Enforcement Context Management
Component 3

Multimodal AI — Vision + URL Crawling

Two additional input modes beyond raw HTML: (1) Live URL crawling via httpx fetches and parses real HTML from any public website. (2) Screenshot input uses LLaMA 4 Scout Vision (Groq) to reconstruct HTML from an uploaded image — enabling accessibility audits with no source code access at all.

Quick start

Get Running in 5 Minutes

  1. Get a free Groq API key at console.groq.com — this is the only external key needed
  2. cp backend/.env.example backend/.env — add both keys
  3. pip install -r backend/requirements.txt
  4. cd backend && python rag/build_kb.py — builds WCAG 2.2 vector store
  5. uvicorn main:app --reload — starts FastAPI on :8000
  6. cd frontend && npm install && npm run dev — starts React on :5173
